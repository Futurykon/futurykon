# Domain Docs

Layout: single-context

- Domain glossary and decisions: `CONTEXT.md` at repo root
- Architectural decision records: `docs/adr/` at repo root (create as needed)

## Consumer rules
- Read `CONTEXT.md` before any task involving domain concepts (scoring, predictions, questions, leaderboard)
- Before creating an ADR, check `docs/adr/` for prior decisions on the same topic
- ADR threshold: only create one when the decision is hard to reverse, surprising without context, and involved genuine trade-offs
